
## Adding 3 Strategic Intelligence Pages to the Audit PDF

### What We're Building

Three new pages will be added to the AI-powered PDF audit report:

1. **Competitor Comparison** — AI-generated health scores, authority gaps, content volume gaps, and page speed gaps for 2-3 real competitors, presented in a table with color-coded badges.
2. **Keyword Opportunity Section** — AI-identified high-value keywords the client could rank for, with monthly search volume estimates, competition level, current rank estimate, and opportunity score shown in a heatmap-style table.
3. **Strategic Growth Forecast** — A 90-day projection table broken into improvement areas (Technical, Content, Conversion) with expected SEO traffic lifts and conversion lifts, plus summary highlight boxes.

All data will be generated by the same AI model already powering the audit, using the real scraped website content — so results will always be specific and realistic for that actual site.

---

### How It Works (No Manual Input Needed)

The existing pipeline:
```text
User enters URL
  → Firecrawl scrapes the site
    → AI (Gemini) analyzes the content
      → Structured JSON returned
        → PDF generated from the data
```

We'll extend the AI prompt to also return competitor data, keywords, and growth forecasts as part of the same JSON response. The AI will derive:
- Competitor names based on industry context found in the scraped content
- Keywords based on the actual products/services described on the site
- Growth forecasts based on current scores and issue severity

---

### Technical Plan

#### Step 1 — Update `AuditResult` type (`src/types/audit.ts`)

Add three new optional fields:

```typescript
competitors?: CompetitorData[];
keywords?: KeywordOpportunity[];
growthForecast?: GrowthForecastItem[];
```

With supporting interfaces:
- `CompetitorData`: name, healthScore, authorityGap, contentVolumeGap, pageSpeedGap
- `KeywordOpportunity`: keyword, monthlySearches, competition, currentRank, opportunity
- `GrowthForecastItem`: area, action, seoLift, conversionLift

#### Step 2 — Extend the AI Prompt (`supabase/functions/analyze-website/index.ts`)

Add three new sections to the analysis prompt asking the AI to return:

```json
"competitors": [
  {
    "name": "CompetitorSite.com",
    "healthScore": 84,
    "authorityGap": "+12",
    "contentVolumeGap": "+15%",
    "pageSpeedGap": "-2s"
  }
],
"keywords": [
  {
    "keyword": "heated dog leash",
    "monthlySearches": 1200,
    "competition": "Medium",
    "currentRank": "Not Ranked",
    "opportunity": "High"
  }
],
"growthForecast": [
  {
    "area": "Technical Fixes",
    "action": "Fix broken links, improve page speed",
    "seoLift": "15-25%",
    "conversionLift": "—"
  }
]
```

The AI will infer competitors from the site's industry/niche, keywords from actual product/service content, and forecast ranges from the current score gaps identified.

#### Step 3 — Pass the new data through the result builder

In the same edge function, extract `competitors`, `keywords`, and `growthForecast` from the parsed AI JSON and include them in the returned `result` object.

#### Step 4 — Add 3 new PDF sections (`src/lib/pdfGenerator.ts`)

After the existing "Implementation Roadmap" section and before "ROI Summary", insert:

**Section 3: Competitor Comparison**
- Section header with page break check
- Intro paragraph explaining the comparison
- `autoTable` with columns: Website | Health Score | Authority Gap | Content Volume Gap | Page Speed Gap
- Client row styled differently (highlighted) vs competitor rows
- Color-coded gap badges: green if competitor is worse, red if better

**Section 4: Keyword Opportunity**  
- Section header
- Intro paragraph
- `autoTable` with columns: Keyword | Monthly Searches | Competition | Current Rank | Opportunity
- Opportunity column color-coded: High = green, Medium = yellow, Low = orange
- Competition column color-coded: Low = green, Medium = yellow, High = red

**Section 5: Strategic Growth Forecast**
- Section header + 90-Day Growth Projection subtitle
- `autoTable` with columns: Improvement Area | Action | Expected SEO Traffic Lift | Expected Conversion Lift
- Summary highlight boxes: "Total SEO Traffic Lift" and "Total Conversion Lift" using filled rounded rectangles matching the existing design style

Existing sections renumbered: Implementation Roadmap → 6, ROI Summary → 7.

### Realistic Output Expectations

Since the AI uses the actual scraped content of the website:
- Competitor names will be real companies in that space (the AI knows the industry from the page content)
- Keywords will be drawn from actual product names, descriptions, and use cases on the site
- Score gaps and growth forecasts will be proportional to the real scores identified in the audit

The result is a fully automated, site-specific strategic report — no manual input required.
